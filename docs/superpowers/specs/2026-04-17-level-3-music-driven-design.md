# 第3关设计规格：音乐驱动轨道生成

## 概述

- **关卡ID**: 3
- **关卡名称**: "Echoes of Earth"（暂定）
- **核心特性**: 根据音乐重音动态生成轨道，重音间隔决定轨道间距
- **目标用户**: 已通关前2关的玩家

---

## 音乐与重音检测

### 输入
- **音乐文件**: `music/BetweenWorlds.mp3`
- **分析起始点**: 第28秒（跳过前28秒的静默/前奏）
- **分析库**: web-audio-beat-detector

### 重音检测
- 提取所有检测到的重音时间点（onset/beat）
- 每个重音对应一个轨道段
- 总重音数 = 轨道总数 = 通关所需跳跃次数

### 轨道间距算法
```
基准节拍间隔 = 0.5秒（假设值，实际由检测结果得出）
基础轨道间距 = SEGMENT_LENGTH(8) + SEGMENT_GAP(0.3) = 8.3

对于每个重音间隔:
  实际间隔 = beatTimes[i+1] - beatTimes[i]
  间距系数 = 实际间隔 / 基准节拍间隔
  轨道间距 = 8.3 * 间距系数

当间隔 < 基准值时: 间距压缩（保持紧凑节奏）
当间隔 > 基准值时: 间距拉大（为球留出缓冲落点的时间）
```

## 轨道系统

### 轨道类型
| 类型 | 概率 | 描述 |
|------|------|------|
| straight | 33% | 单条完整轨道，最简单 |
| double | 33% | 两条分离轨道，需左右选择 |
| triple | 33% | 三条分离轨道，难度最高 |

### 颜色逻辑（与前2关一致）
- **straight**: 单色（pathColor）
- **double**: pathColor + 另一随机颜色，随机排列左右
  - 确保一个颜色与 pathColor 匹配（保证有安全落点）
- **triple**: 三色随机排列
  - 颜色使用 `shuffleColors(3)` 随机打乱

### pathColor 刷新规则
- 每次生成 straight 时，pathColor 刷新为新随机色
- double/triple 使用当前 pathColor

## 游戏物理与流程

### 初始状态
1. 游戏开始
2. 球从 `y = 8` 的高处自然落下
3. 前2秒留空（28秒前的缓冲时间在预加载时处理）
4. 音乐从28秒位置开始播放
5. 当球落到第一个轨道上时，正式开始跳跃计数

### 轨道移动
- 轨道以 `sharedVelocity` 向 +Z 方向移动
- 速度由第一个轨道的间距计算得出

### 胜利流程
1. 球跳完最后一个轨道
2. 在最后一个轨道后方生成黑洞
3. 黑洞向球移动并吸入
4. 显示胜利界面

---

## UI 显示

### 右上角距离计数器
- **正常状态**: 显示 "XXXm"（已前进的距离）
- **终点临近状态**: 当最后一个轨道出现在屏幕上时
  - 切换为 "XXXm"（剩余距离）
  - 实时刷新直到胜利

### 距离计算
```
已前进距离 = 轨道总长度 - 最后一个轨道的Z位置
剩余距离 = 最后一个轨道的Z位置 - 黑洞Z位置
```

---

## 黑洞效果

### 生成时机
- 当球完成最后一个轨道的跳跃时
- 在最后一个轨道后方（+Z方向）生成黑洞

### 外观与行为
- 使用现有 blackHole 着色器
- 向球方向移动并吸入
- 触发后游戏状态变为 'victory'

---

## 音乐同步

### 播放控制
- 游戏开始时：从第28秒开始播放音乐
- 游戏结束时：停止音乐播放

### 实现方式
- 使用 `HTMLAudioElement.currentTime = 28` 跳转
- 或使用 Web Audio API 的 startTime 参数
